Why Bulma?

It's pretty neat

  • Benefits
    • Very simple but comprehensive
    • Based on flex-box which is successor to old-fashioned CSS
    • Can work alongside other CSS frameworks because it does not style unclassed objects (must remove "reset" css)
    • Minimal: Learn it in a day
    • Incredibly responsive
    • Not opinionated: We can customize it easily
    • Competitive with larger, more opinionated frameworks like Bootstrap and Semantic-UI
    • CSS-only, no built-in JS framework requirements to adapt or integrate
    • Does not look like Bootstrap
  • Drawbacks
    • Default for 3rd party packages is Bootstrap which is used everywhere. Each new integration requires a bit of simplification and removing of excess markup to get it ready for Bulma
    • No built-in JS means some JS coding for UI/UX (turning out to be a good thing so far -- wf 201703)
    • Flexbox approach means some SCSS mixins or other libraries for older CSS won't work as expected
